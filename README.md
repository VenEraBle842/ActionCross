**Action Cross** — 2D-аркада на мотоцикле с физикой на основе интеграции Верле.
Игрок проезжает холмистый ландшафт, собирает яблоки и избегает шипов.

> Написана на **C++20** без сторонних физических библиотек.

---

## Геймплей

1. Разгоняйте мотоцикл по холмистому рельефу
2. Собирайте все яблоки, разбросанные по уровню
3. Избегайте шипов — столкновение означает аварию
4. Доберитесь до финиша (цветок активируется после сбора всех яблок)

Цель — пройти уровень за минимальное время.

## Ключевые особенности

### Физика на основе интеграции Верле (Verlet Integration)

В отличие от классического подхода Эйлера (`pos += vel * dt`), здесь используется безскоростная интеграция Верле:

$$
newPos = 2 \cdot pos - oldPos + acceleration \cdot dt^2
$$

Скорость неявно выражена через разницу `pos − oldPos`. Это дает:
- **Стабильность** при высоких нагрузках и жестких ограничениях
- **Определеность** — результат не зависит от порядка применения сил
- **Простоту** — не нужно хранить скорость как отдельную величину

### Кулоновское трение

Модель трения при контакте с землей разделяет скорость на нормальную и тангенциальную компоненты и применяет формулу Кулона:

$$
F_{\text{трения}} \le \mu \cdot |F_{\text{нормальная}}|
$$

Реализовано **статическое** (полная остановка при малых скоростях) и **кинетическое** (пропорциональное замедление) трение. Дополнительно реализовано противодействие гравитационному скатыванию на склонах: если тангенциальная компонента гравитации попадает в конус статического трения, она полностью компенсируется.

### Пространственное хеширование (Spatial Hashing)

Для оптимизации обнаружения столкновений мир делится на сетку ячеек (`cellSize = 64px`). Каждый отрезок ландшафта вставляется в ячейки, которые покрывает его параллельный осям ограничивающий параллелепипед (Axis-Aligned Bounding Box, AABB). При проверке столкновений для частицы запрашиваются только ячейки вокруг нее — это сводит сложность с `O(n)` до `O(1)` в среднем случае.

### Система ограничений (Constraints)

Мотоцикл смоделирован как граф из 4 частиц, соединенных 6 ограничениями-стержнями с настраиваемой жесткостью (0..1) и демпфированием. Ограничения решаются итеративно (8 проходов за физический шаг) методом проекции позиций — каждая частица сдвигается пропорционально обратной массе.

### Фиксированный шаг физики

Физика работает с постоянным временным шагом `1/120 с`, а не с `deltaTime` рендеринга. Используется аккумулятор:

```cpp
accumulator += dt;
while (accumulator >= PHYSICS_DT) {
    physics.Step(PHYSICS_DT);
    accumulator -= PHYSICS_DT;
}
```

Это гарантирует одинаковое поведение при любой частоте кадров и предотвращает «спираль смерти» (frame dt ограничен 50 мс).

### Поведение в воздухе

Физика корректно различает состояние «на земле» и «в воздухе»:
- **Газ в воздухе** — нет сцепления, только раскрутка заднего колеса (визуальная угловая скорость)
- **Тормоз в воздухе** — замедляет только вращение колес, не влияет на траекторию
- **На земле** — тяга передается по касательной к реальной поверхности, с учетом 3-го закона Ньютона

### Собственная библиотека контейнеров

В проекте используется `MutableArraySequence` из библиотеки [ADTLinearOrganization](https://github.com/VenEraBle842/ADTLinearOrganization.git) — кастомная реализация динамического массива с интерфейсом АТД «последовательность».

## Физический движок

Каждый шаг `PhysicsWorld::Step(dt)` выполняет 4 фазы:

1. **Накопление сил** — гравитация (`980 px/s²` вниз) прикладывается ко всем частицам
2. **Интеграция Верле** — обновление позиций с демпфированием (`0.999`)
3. **Решение ограничений** — 8 итераций для стабилизации жестких связей
4. **Обнаружение и разрешение столкновений** — круг <-> отрезок линии:
   - широкая фаза через `SpatialHash`
   - узкая фаза: проекция точки на отрезок, проверка проникновения
   - выталкивание, разделение скорости на компоненты, кулоновское трение

## Модель мотоцикла

Мотоцикл состоит из 4 частиц-узлов:

| Узел              | Масса | Радиус | Назначение                              |
|:------------------|:-----:|:------:|:----------------------------------------|
| Переднее колесо   | 1.8   | 18 px  | Ведомое колесо, коллизии с землей       |
| Заднее колесо     | 1.8   | 18 px  | Ведущее колесо, передача тяги           |
| Корпус/шасси      | 1.0   |  0 px  | Инерция; без коллизий                   |
| Голова водителя    | 0.5   |  8 px  | Касание земли = мгновенная смерть       |

6 ограничений формируют жесткий каркас:

```
Переднее ––– (рама, 0.9) ––– Заднее
    │ \                        / │
    │  (вилка, 0.7)  (маятник, 0.7)
    │         \      /         │
    │          Корпус          │
    │            │             │
    │       (шея, 0.5)         │
    │            │             │
 (распорка А)  Голова  (распорка Б)
     (0.3)                (0.3)
```

Центр масс вычисляется как взвешенное среднее позиций всех узлов и используется для позиционирования камеры.

## Система уровней

Уровень определяется:
- **Вершинами полигона** — последовательность точек, которая автоматически замыкается и разбивается на отрезки (`BuildSegments`)
- **Точкой спавна** — начальная позиция мотоцикла
- **Яблоками** (`Collectible`) — собираются при сближении с любым узлом байка
- **Шипами** (`Hazard`) — при контакте с любым узлом = `CRASHED`
- **Финишем** (`Finish`) — активируется только после сбора всех яблок

## Камера

Реализована 2D-камера с **экспоненциальной интерполяцией** (exponential lerp):

```cpp
float t = 1.0f - exp(-smoothness * dt);
position = lerp(position, target, t);
```

Это обеспечивает плавное, физически корректное слежение за мотоциклом — скорость сближения пропорциональна расстоянию до цели и не зависит от частоты кадров.

## Сборка и запуск

### Требования

- Компилятор C++20 (MSVC, GCC 10+, Clang 12+)
- CMake 3.14+
- Windows (текущая конфигурация линковки: `user32`, `gdi32`, `opengl32`, `gdiplus`, `Shlwapi`, `dwmapi`)

### Сборка из командной строки

```bash
# Клонирование репозитория
git clone https://github.com/VenEraBle842/ActionCross.git
cd ActionCross

# Конфигурация и сборка
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release
```

### Сборка в CLion

1. Откройте папку проекта в CLion
2. CLion автоматически подхватит `CMakeLists.txt`
3. Дождитесь загрузки зависимостей (первая конфигурация)
4. Нажмите **Build** -> **Run**

### Запуск

```bash
./build/Release/ActionCross.exe
```

Окно откроется с разрешением **800×600** пикселей.
